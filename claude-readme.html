Claude Code Install Guide

Get Dev.IQ running in Claude Code in under 20 minutes — personal trial, invisible to your team until you're ready.

What you need

RequirementNotes
Claude Code CLIInstalled and authenticated (claude --version should work)
Gitgit --version in a terminal should return a result
The dev-iq zipProvided by your engagement lead

Node.js 18+ is optional — it's needed for live MCP data pull from ADO/GitHub. Skills fall back gracefully to paste-mode without it. You can wire MCP later.

Install in five steps

Extract the zip

Unzip to a permanent location on your machine. This is the pack source — it stays here and you bootstrap from it into your repos.

# macOS / Linux — example locations
~/tools/dev-iq/
/opt/dev-iq/

# Windows — example locations
C:\Tools\dev-iq\
Windows only — unblock extracted files

Files from a downloaded zip are flagged by Windows and may be blocked from running. Run this once after extracting:

Get-ChildItem "C:\Tools\dev-iq" -Recurse | Unblock-File

Run bootstrap in trial mode

Navigate to your target repo (the codebase you want Dev.IQ to analyse) and run bootstrap from the extracted pack:

# macOS / Linux
cd ~/code/my-repo
bash ~/tools/dev-iq/scripts/bootstrap.sh --mode=trial --preset=solo

# Windows PowerShell
cd C:\code\my-repo
powershell -File "C:\Tools\dev-iq\scripts\bootstrap.ps1" -Mode trial -Preset solo

When prompted "Just you, or the whole team?" — answer [1] Just me (solo). Trial mode means all pack files are hidden from git via .git/info/exclude. Your .gitignore is never touched, nothing is committed, nothing reaches your teammates.

Fill in the config

Open .dev-iq/config.yaml in the repo and fill in what you know:

client:
  name: "[your client or project name]"

tracker:
  type: "ado"   # or "jira"
  ado:
    org_url: ""    # e.g. https://dev.azure.com/your-org
    project: ""    # e.g. YourProjectName

vcs:
  type: "github"  # or "ado-repos", "gitlab", "bitbucket"

stack:
  languages:
    - ""           # e.g. typescript, csharp, python

Leave anything you're unsure of blank — skills still work, they'll ask you for details inline when needed.

Verify it's working

Open the repo in a terminal. Type / to see available skills — you should see the Dev.IQ skills listed. Then run:

/explain-code

Point the skill at any source file. If the response includes a Purpose section and an INTENT signal verdict — you're live.

Expected pass criteria

Response contains: a Purpose summary, one or more DI signal assessments (INTENT / DESIGN / QUALITY / RISK), and a recommendation. If you see all three — Dev.IQ is running correctly.

Skills appear but hooks are silent?

Open .claude/settings.json in the repo and confirm the hooks section is present. If it's missing, re-run bootstrap. If present but hooks still don't fire, confirm you are on Dev.IQ v0.11.0 — earlier versions had incorrect hook event names.

Try a real skill on your work

Open a branch with recent changes and run:

/review-pr-readiness

You should get a four-layer DI assessment — INTENT, DESIGN, QUALITY, RISK — with a Go / Hold / Discuss recommendation. This is the core of what Dev.IQ does.

Using the Dev-IQ agents

Dev.IQ ships two Claude Code subagents. Both are discovered automatically from .claude/agents/.

AgentWhen to use
dev-iqDefault. Full tools, full execution. Pick this for code review, PR readiness, security review, debugging, scaffolding — anything where you want results.
dev-iq-planRead-only planning mode. Produces a structured DI four-layer plan for complex or risky changes. Ends with a Start Implementation handoff to dev-iq. Use when you want to think through a change before touching any files.

Both agents are maturity-aware: they adjust recommendations based on maturity.tier in .dev-iq/config.yaml and proactively raise traceability gaps, design issues, and governance concerns.

What bootstrap wired (and what it didn't)

Bootstrap installs every file to disk, but a few surfaces only load when they are in the right place. If something feels off, check these first:

SurfaceWhere it must liveWhat breaks if missing
CLAUDE.mdRepo rootDev-IQ has no DI reasoning layer — behaves like generic Claude
.claude/agents/dev-iq.md.claude/agents/Dev-IQ subagent not available
.claude/skills/.claude/Skill invocations (/review-code etc.) fail to load
.dev-iq/config.yamlRepo rootSkills can't read your stack config or maturity tier
.claude/settings.jsonRepo rootHindsight Hooks don't fire at session start/end

Bootstrap creates all of these during install. If one is missing, re-run bootstrap with --preset=solo — it is safe to run multiple times.

Graduate to team adoption

When you're ready to share Dev.IQ with the rest of the team, graduate from trial mode to committed:

# macOS / Linux
bash ~/tools/dev-iq/scripts/bootstrap.sh --graduate

# Windows
powershell -File C:\Tools\dev-iq\scripts\bootstrap.ps1 -Graduate

# Commit the pack files
git add .dev-iq .claude .github CLAUDE.md AGENTS.md
git commit -m "chore: adopt Dev.IQ agent pack"

Graduating removes the .git/info/exclude block and makes the files visible to git. From here, teammates clone the repo and get Dev.IQ automatically.

Removing the pack

# macOS / Linux
bash ~/tools/dev-iq/scripts/bootstrap.sh \
  --target ~/code/my-repo \
  --uninstall

# Windows
powershell -File C:\Tools\dev-iq\scripts\bootstrap.ps1 \
  -Target C:\code\my-repo \
  -Uninstall

Uninstall reads the install manifest and restores any files that existed before. Files you edited post-install (config.yaml, governance.md) are saved as <file>.di.uninstall-saved so nothing is silently lost.

Security — what stays on your machine

  • No source code or repository content is sent anywhere beyond what Claude Code already sends to Anthropic when you use it normally.
  • ADO/GitHub credentials are stored in your OS keychain by VS Code — never written to any file, never committed.
  • The filesystem MCP server only reads files inside the repo folder you have open. It cannot access anything outside that folder.
  • Trial mode files are local-only. Nothing is committed to the shared repo.
  • See .dev-iq/governance.md for the full data boundary statement.
