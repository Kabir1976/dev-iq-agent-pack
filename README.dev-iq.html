Dev.IQ — Full Reference

The complete reference for the Dev.IQ Agent Pack — install options, DI signal model, full skill registry, maturity tiers, MCP wiring, governance, and troubleshooting.

At a Glance

Dev.IQ is a set of markdown, YAML, and JSON files that bootstrap into a repo and give GitHub Copilot Chat and Claude Code a Developer Intelligence reasoning layer — 27 skills, two agents, and MCP wiring for ADO and GitHub — without deploying any runtime or service.

Fastest path to first value: extract the zip → run bootstrap.sh --preset=solo against your repo → open Copilot Chat → select Dev-IQ → run /review-pr-readiness on a branch with changes. Under 20 minutes, requires nothing beyond VS Code and Git.

Skills work without MCP.

If Node.js isn't installed or credentials aren't configured, every skill that needs external data falls back to asking you to paste the content inline. Nothing is blocking.

The DI Signal Model

Every skill in this pack reasons through four DI signal layers in order, then synthesizes them into Decision Confidence:

INTENT

What are we building? Does the code match the requirement and acceptance criteria?

DESIGN

Is it built the right way? Patterns, architecture, naming, abstractions in check?

QUALITY

Is it production-ready? Error handling, security, test coverage, performance?

RISK

What could break? Dependencies, schema changes, blast radius, breaking contracts?

Each signal is assessed as STRONG (positive evidence), WEAK (gap or risk identified), or UNGRADED (data unavailable). UNGRADED is never treated as safety — it blocks a confident verdict.

Layer threshold actions by maturity tier

TierGreenYellowRed
EarlyProceed with AI guidanceHuman review + coaching noteBlock + coaching report
MidAuto-proceedHuman review requiredBlock + diagnostic report
HigherFully autonomousAuto-assign senior reviewerBlock + auto-rollback suggestion

Enterprise Governance: Agent Autonomy Classification

Dev.IQ operates at the Advise and Act with Approval tiers of the Gartner AI agent autonomy model. No skill operates at Act Autonomously.

Gartner TierWhat the agent doesDev.IQ skills at this tier
ObserveReads and surfaces information. No recommendations./explain-code, /identify-dependencies, /generate-traceability
AdviseMakes recommendations. Human decides and acts./review-code, /review-security, /review-pr-readiness, /review-architecture, /validate-acceptance-criteria, /refactor-code (plan phase), /blast-radius-estimator, /estimate-effort, /review-observability, /review-ai-integration
Act with ApprovalProduces a complete artifact. Developer reviews and applies manually./scaffold-feature, /generate-adr, /design-data-model, /generate-rollback-plan, /generate-release-notes, /debug-issue, /review-deployment-readiness, /refactor-code (apply phase), /generate-openapi, /onboard-codebase, /dev-iq-tailor

The @di-review-required marker is mandatory.

Every skill output carries this marker. It is a formal signal that AI-generated content has not been human-reviewed yet — the audit trail for the Act-with-Approval tier. Not optional.

Installation

The pack supports two install paths. Pick based on whether you're evaluating Dev.IQ or deploying it into a team's repo.

Path A — Try it on the pack folder itself

Best when you want to explore Dev.IQ without touching your team's repository. Extract the pack zip, then run bootstrap from inside the pack folder.

# macOS / Linux
cd path/to/dev-iq
bash scripts/bootstrap.sh --preset=solo

# Windows PowerShell
cd C:\Tools\dev-iq
powershell -File scripts\bootstrap.ps1 -Preset solo

Idempotent on every platform — safe to re-run. Reverse with --uninstall.

Path B — Install into your codebase

This is the real deployment path. Run from a terminal in your target repo — no editor required:

# 1. Extract the pack zip somewhere on your machine (one time, anywhere)
#    e.g. ~/tools/dev-iq  or  C:\Tools\dev-iq on Windows

# 2. cd into YOUR repo
cd ~/code/my-app

# 3. Run bootstrap pointing at the extracted pack
bash ~/tools/dev-iq/scripts/bootstrap.sh --mode=trial

# Windows PowerShell
powershell -File C:\Tools\dev-iq\scripts\bootstrap.ps1 -Mode trial

Already have the pack loaded in your editor? Run /dev-iq-bootstrap from Copilot Chat or Claude Code — same outcome, chat-driven prompts.

Distributing to Colleagues

Option 1 — Zip file (no internet required)

cd /path/to/dev-iq
git archive --format=zip --output=~/Desktop/dev-iq-v0.11.0.zip HEAD

git archive includes only committed files — no .git folder, no machine-specific files. The result is exactly what a colleague needs to run bootstrap. Colleagues follow the trial install guide.

Option 2 — GitHub collaborator

Invite them to the GitHub repo: Settings → Collaborators → Add people. They clone once, then run bootstrap against their own repos.

Option 3 — Internal Git server (enterprise long-term)

git remote add internal https://your-internal-git/dev-iq.git
git push internal main

Upgrading

  1. Read the release notes for what changed and any migration notes.
  2. Uninstall the current path: bash scripts/bootstrap.sh --uninstall in each installed location.
  3. git pull (or re-extract the zip) to the new version.
  4. Re-run the same install path to refresh all surfaces.

Maturity Awareness

The pack reads .dev-iq/maturity-profile.md and adjusts behavior. Set during the DI Diagnostic and re-evaluated quarterly.

TierBehavior
EarlyFoundation + intent + design review only. Risk assessment in advisory mode. All outputs are drafts with coaching notes. Human review required for every output.
MidAdd quality signals, automated code review, PR readiness in suggest-only mode. DI routing operates as designed. Risk assessment provides structured reports.
HigherFull pack including blast radius estimation, autonomous PR readiness verdict, and predictive deployment risk. Decision Confidence signal available in Phase 2.

Adding a New Skill

  1. Create .github/skills/your-skill/SKILL.md with YAML frontmatter: name, description, di_signal, maturity_required.
  2. Add sections: Overview, When to Use, Instructions, Inputs Required, Output Format, Governance.
  3. Update the skill registry in docs/reference.md.
  4. Claude Code picks up the skill automatically via the .claude/skills/ symlink — no second copy needed.

Troubleshooting

Skill can't find the work item

Confirm MCP is wired (.vscode/mcp.json) and the PAT has read access to Work Items. Test with a minimal MCP query directly. If Node.js is not installed, skills will prompt you to paste the work item content manually — that is the expected fallback.

Generated code doesn't match our conventions

Update .github/instructions/di-code-standards.instructions.md with explicit examples of your project's patterns. The agent reads instructions before generating code.

Dev-IQ does not appear in the agent dropdown

Confirm .github/agents/Dev-IQ.agent.md exists in the workspace root. In Visual Studio 2022, also confirm you enabled "Enable repository custom instructions" in Tools → Options → GitHub → Copilot → (General).

Skill outputs feel generic

The pack relies on instruction files for project-specific context. Update di-code-standards.instructions.md and di-traceability.instructions.md with real examples from your codebase, and fill in .dev-iq/telemetry-overlay.md so the agent knows your actual tool stack.

Hooks are silent

Open .claude/settings.json in the workspace and confirm the hooks section is present. If missing, re-run bootstrap. If present but hooks still don't fire, confirm you are on v0.11.0 — earlier versions had incorrect hook event names that caused silent failures.

Risk assessment feels off

Adjust signal weighting in .github/skills/review-pr-readiness/SKILL.md. Each DI layer is independently tunable. Also verify .dev-iq/telemetry-overlay.md maps to your actual data sources.

What Dev.IQ Is Not

  • It is not a runtime. There is no service to deploy.
  • It is not a SaaS. The client owns the files — if the consulting team rotates off, the pack stays.
  • It is not a replacement for engineering judgment. Every output is a draft; human review is required.
  • It is not a replacement for Assert.IQ. Use both packs together for full SDLC + QE coverage.
  • It is not a tooling pitch. Use it where the maturity supports it. Lead with DI thinking, not with this pack.
