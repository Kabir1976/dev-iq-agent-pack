Windows checklist — run this before anything else

These are the most common blockers on corporate Windows machines. Run through these first.

1. PowerShell execution policy

Get-ExecutionPolicy -Scope CurrentUser
# If Restricted or AllSigned:
Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned

If you get "Access denied", your machine is managed by Group Policy — speak to your IT contact before proceeding.

2. Unblock zip contents

Get-ChildItem "C:\Tools\dev-iq" -Recurse | Unblock-File

3. Developer Mode (recommended, not required)

Settings → Privacy & security → For developers → Developer Mode On. Enables directory symlinks without an admin shell. Without it, bootstrap falls back to copying skill files — safe but requires re-running after edits.

4. Check your IDE version

VS Code: Help → About — requires 1.99 or later + Copilot Chat v0.22+. Visual Studio 2022: Help → About — requires v17.14 or later. Update before continuing if behind.

Install in five steps

Extract the zip

Unzip to a permanent location. This is the pack source — it stays here and you bootstrap from it into your repos.

# Recommended locations
C:\Tools\dev-iq\       # Windows
~/tools/dev-iq/        # macOS / Linux

Then run the Unblock-File command above if on Windows.

Run bootstrap in trial mode

Open PowerShell (Windows) or Terminal (macOS/Linux) and run bootstrap, pointing it at your target repo:

Windows
macOS / Linux
# Windows PowerShell
powershell -File "C:\Tools\dev-iq\scripts\bootstrap.ps1" `
  -Target "C:\code\my-repo" `
  -Preset solo
# macOS / Linux
bash ~/tools/dev-iq/scripts/bootstrap.sh \
  --target ~/code/my-repo \
  --preset=solo \
  --mode=trial

When prompted "Just you, or the whole team?" — answer [1] Just me (solo).

Trial mode = invisible to git.

All pack files land in your workspace but are hidden from git via .git/info/exclude. Your .gitignore is never touched, nothing is committed, nothing reaches your teammates until you graduate to committed mode.

Fill in the config

Open .dev-iq/config.yaml in VS Code and fill in what you know:

client:
  name: "[your client or project name]"

tracker:
  type: "ado"   # or "jira"
  ado:
    org_url: ""    # e.g. https://dev.azure.com/your-org
    project: ""    # e.g. YourProjectName

vcs:
  type: "ado-repos"  # or "github", "gitlab", "bitbucket"

stack:
  languages:
    - ""           # e.g. csharp, typescript, java

Leave anything you're unsure of blank — skills still work, they'll ask you for details inline.

Wire your ADO credentials (optional but recommended)

This lets skills pull live work item and PR data from ADO without you having to paste it.

Create a Personal Access Token in ADO

ADO → top-right avatar → Personal access tokens → New Token

Scopes needed: Work Items (Read), Code (Read), Pull Request Threads (Read & Write)

Connect in VS Code

Open Copilot Chat. VS Code will prompt for ADO_ORG_URL and ADO_PAT the first time a skill needs them. Enter your org URL and the token. VS Code stores both in your OS keychain — you won't be asked again.

Node.js required for MCP.

MCP servers require Node.js 18+. If you don't have it, skip this step — skills fall back to paste mode automatically. You can wire MCP later once Node.js is available.

Verify it's working

Open the target repo in VS Code. Open Copilot Chat (Ctrl+Alt+I). Look for Dev-IQ in the agent dropdown at the top of the chat panel. Select it, then run:

/explain-code

Point the skill at any source file. If the response includes a Purpose section and an INTENT signal verdict — you're live.

Expected pass criteria

Response contains: a Purpose summary, one or more DI signal assessments (INTENT / DESIGN / QUALITY / RISK), and a recommendation. If you see all three — Dev.IQ is running correctly.

Dev-IQ does not appear in the agent dropdown?

This is the most important thing to report back if it happens. Check that .github/agents/Dev-IQ.agent.md exists in the workspace. If you are using Visual Studio 2022, also see the next section. Note your IDE version and Copilot extension version (Help → About) and send that to the person who gave you this guide.

Visual Studio 2022 — one-time setup

This step is easy to miss and blocks everything if skipped.

In Visual Studio: Tools → Options → GitHub → Copilot → (General)

Turn on "Enable repository custom instructions" and click OK.

Without this, the DI reasoning layer never loads.

copilot-instructions.md is the always-on file that gives Dev-IQ its DI reasoning. Without this VS option enabled, the file is ignored and Dev-IQ responds like a generic Copilot agent with no DI context.

To invoke the Dev-IQ agent in Visual Studio, type @Dev-IQ in Copilot Chat — it should autocomplete. Then type /explain-code to verify.

Using the Dev-IQ agents

Dev.IQ ships two custom Copilot Chat agents. Select them from the agent dropdown at the top of the chat panel.

AgentWhen to use
Dev-IQDefault. Full tools, full execution. Pick this for code review, PR readiness, security review, debugging, scaffolding — anything where you want results.
Dev-IQ-PLANRead-only planning mode. Produces a structured DI four-layer plan for complex or risky changes. Ends with a Start Implementation handoff button to Dev-IQ for execution. Use when you want to think through a change before touching files.

Both agents are maturity-aware: they adjust recommendations based on maturity.tier in .dev-iq/config.yaml and proactively raise traceability gaps, design issues, and governance concerns.

Graduate to team adoption

When you're ready to share Dev.IQ with the rest of the team:

# macOS / Linux
bash ~/tools/dev-iq/scripts/bootstrap.sh --graduate

# Windows
powershell -File C:\Tools\dev-iq\scripts\bootstrap.ps1 -Graduate

# Commit the pack files
git add .dev-iq .claude .github CLAUDE.md AGENTS.md
git commit -m "chore: adopt Dev.IQ agent pack"

Teammates who clone the repo will automatically get Dev.IQ instructions and skills loaded into their Copilot Chat and Claude Code sessions.

Removing the pack

# Windows
powershell -File "C:\Tools\dev-iq\scripts\bootstrap.ps1" `
  -Target "C:\code\my-repo" `
  -Uninstall

# macOS / Linux
bash ~/tools/dev-iq/scripts/bootstrap.sh \
  --target ~/code/my-repo \
  --uninstall

Uninstall reads the install manifest and restores any files that existed before the pack was installed. Files you edited post-install (config.yaml, governance.md) are saved as <file>.di.uninstall-saved so nothing is silently lost.

Security — what stays on your machine

  • No source code or repository content is sent anywhere beyond what GitHub Copilot already sends when you use Copilot Chat normally.
  • ADO/GitHub credentials are stored in the Windows Credential Manager or macOS Keychain — never written to any file, never committed.
  • The filesystem MCP server only reads files inside the repo folder open in VS Code. It cannot access anything outside that folder.
  • Trial mode files are local-only. Nothing is committed to the shared repo.
  • See .dev-iq/governance.md for the full data boundary statement.
