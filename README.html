The shift from development to Developer Intelligence

Traditional development asks one question: does this work? Developer Intelligence asks four — and synthesizes them into Decision Confidence no single linter pass or green build can give you.

LAYER 01

INTENT

Are we building the right thing? Does the code match the requirement and acceptance criteria?

LAYER 02

DESIGN

Is it built the right way? Patterns, architecture, naming, and abstractions in check?

LAYER 03

QUALITY

Is it production-ready? Error handling, security, test coverage, and performance validated?

LAYER 04

RISK

What could break? Dependencies, schema changes, blast radius, and breaking-change exposure mapped.

The immediate impact

  • Developers stop asking "does it compile?" and start asking "does this solve the right problem the right way?"
  • PR conversations shift from gut-feel to evidence.
  • Design decisions get documented and traceable — not just implied.
  • Architecture guardrails enforce themselves through every code review, every PR, every deployment check.

Get started in three steps

Install the pack

Two install paths. Pick the one that matches how comfortable you are dropping the pack into your team's codebase.

Path A · Try it on the pack

Best if you want to explore Dev.IQ before touching your team's repository. Extract the zip, run bootstrap from inside the pack folder.

# macOS / Linux
cd path/to/dev-iq
bash scripts/bootstrap.sh --preset=solo

# Windows PowerShell
cd C:\Tools\dev-iq
powershell -File scripts\bootstrap.ps1 -Preset solo

Reverse with --uninstall.

Path B · Install into your codebase

The real deployment path. Run bootstrap from a terminal in your target repo — no editor required:

# 1. Extract the pack zip somewhere (one time, anywhere)
#    e.g. ~/tools/dev-iq  or  C:\Tools\dev-iq on Windows

# 2. cd into YOUR repo
cd ~/code/my-app

# 3. Run bootstrap pointing at the extracted pack
bash ~/tools/dev-iq/scripts/bootstrap.sh --mode=trial

# Windows PowerShell
powershell -File C:\Tools\dev-iq\scripts\bootstrap.ps1 -Mode trial

The script is fully standalone — accepts --preset=solo|pod, prompts interactively, and writes everything Copilot and Claude need. No chicken-and-egg. You do not need to open VS Code or Claude Code first.

What does --mode=trial do?

Trial hides pack files from git via .git/info/exclude — only you see them; your codebase .gitignore is never touched. Graduate to committed any time with --graduate.

Already have the pack loaded in your editor? Run /dev-iq-bootstrap from Copilot Chat or Claude Code — same outcome, chat-driven prompts.

Bootstrap surface overview

Bootstrap writes twelve surfaces: .dev-iq/, .github/instructions/, .github/copilot-instructions.md, .github/skills/, .github/agents/, .claude/agents/, .claude/skills (symlink), .claude/settings.json, CLAUDE.md, AGENTS.md, .vscode/settings.json + .vscode/mcp.json, and hooks/. SHA256-compares before writing — pre-existing files are preserved, never silently overwritten.

Configure for your codebase

Bootstrap places the pack; configuration tailors it to your project.

  • .dev-iq/maturity-profile.md — Set your maturity tier: Early, Mid, or Higher. Agents read this on every code and delivery question and scale behavior accordingly.
  • .dev-iq/governance.md — Set your compliance posture: constraints the agents must respect and never override.
  • .dev-iq/config.yaml — Set client.name, tracker.type (ado or jira), vcs.type, and language. Every skill reads this.
  • .dev-iq/telemetry-overlay.md — Map each DI signal to your actual data sources: ADO/Jira for intent, SonarQube/Codecov for quality, your dependency manifest for risk. Or attach it to chat and say: "Customize this overlay for my stack."
  • .vscode/mcp.json — Credentials are entered when VS Code prompts (stored in OS keychain, never committed). See .vscode/MCP.md for per-server setup.

Upgrade

Upgrades are explicit and intentional.

  1. Read the Releases page for what changed and any migration notes.
  2. Uninstall the current path you used:
    • Path A (pack-as-workspace): bash scripts/bootstrap.sh --uninstall
    • Path B (codebase install): bash scripts/bootstrap.sh --uninstall in each target repo
  3. git pull (or re-extract the zip) to the new version, then re-run the same path to refresh.
  4. Re-run /dev-iq-bootstrap to refresh workspace surfaces.
